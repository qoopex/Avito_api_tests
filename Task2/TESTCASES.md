# Тест-кейсы для автотестов API


# Тест-сьют "Создание объявления"

## TC1.1: Создание объявления c корректными данными
### Цель: 
Проверить создание нового объявления.

### HTTP-Метод: 
POST

### Эндпоинт:
	/api/1/item

### Предусловия:  
Уникальный SellerId.

### Шаги воспроизведения:
1. Отправить POST-запрос с корректными данными(sellerID, name, price, statistics).

### Ожидаемый результат:
- Код состояния 200.
- В теле ответа есть id объявления.

## TC1.2: Создание объявления с некорректными данными
### Цель: 
Проверить обработку ошибок при некорректных данных.

### HTTP-Метод: 
POST

### Эндпоинт:
	/api/1/item

### Предусловия:
Некорректные данные.

### Шаги воспроизведения:
1. Отправить POST-запрос с некорректными данными.

### Ожидаемый результат:
- Код состояния 400.
- В теле ответа сообщение об ошибке.


# Тест-сьют "Получение объявлений"

## TC2.1: Получение объявления по ID
### Цель:
Проверить возможность получения объявления по айди.

### HTTP Метод: 
GET

### Эндпоинт:
	/api/1/item/:id

### Предусловия:
Объявление с данным id заранее создано.

### Шаги воспроизведения:
1. Отправить GET-запрос с существующим id.

### Ожидаемый результат:
- Код состояния 200.
- Тело ответа содержит информацию о запрошенном объявлении.

## TC2.2: Получение несуществующего объявления

### Эндпоинт:
	/api/1/item/:id
	
### Предусловия:
id не существует.

### Шаги воспроизведения:
1. Отправить GET-запрос с несуществующим id.

### Ожидаемый результат:
-Код состояния 400.

## TC2.3: Получение всех объявлений пользователя
### Цель:
Проверить получение всех объявлений конкретного пользователя.

### HTTP Метод: 
GET

### Эндпоинт:
	/api/1/:sellerID/item
	
### Предусловия:
У пользователя есть хотя бы одно объявление.

### Шаги воспроизведения:
1. Отправить GET-запрос с sellerID.

### Ожидаемый результат:
- Код состояния 200.
- Тело ответа содержит список объявлений пользователя с sellerID.

# Тест-сьют "Удаление объявлений"

### Цель:
Проверить удаление объявления.

### HTTP Метод:

### Эндпоинт:
	/api/2/item/:id
	
### Предусловия:
Объявление с данным id существует.

### Шаги воспроизведения:
1. Отправить DELETE-запрос с id.
2. Попробовать GET-запрос к тому же id.

### Ожидаемый результат:
- Код состояния 200 при DELETE-запросе.
- Код состояния 400 при GET-запросе.
